I've been a DragonFly BSD committer since mid 2009, and have since contributed to several parts of the OS, mostly to the kernel and drivers as can be seen below. If you want to contact me regarding anything DragonFly related, I suggest you do so by sending me an email to alexh (at) dragonflybsd (dot) org or by using the Contact Form. 

If you want me to work on a specific area or require some functionality that is currently not present, you can sponsor me to do so; contact me for further details.

Contribution Description Release
DevFS Device File System 2.3
Active disk probing probes for on-disk metadata on connection of device, instead of on open 2.3
Unix98 pty support   2.3
Writing/updating/porting several drivers (e.g. hifn, glxsb, cs5536, nsclpcsio, kbdmux, vn, ...) 2.5
Kernel minidump support ported from FreeBSD 2.5
GPIO framework a general purpose I/O framework, modelled after OpenBSD's 2.5
Watchdog Framework a watchdog framework, modelled after OpenBSD's 2.5
Linux Emulation (Linuxulator) updated to linux kernel 2.6 syscalls 2.5
dsched an I/O Scheduler Framework, allowing for pluggable I/O scheduling policies 2.7
dsched FQ policy a dsched fair queueing policy 2.7
proplib port from NetBSD 2.7
dm device mapper, port from NetBSD, including lvm2 userland part 2.7
dm crypt target A disk encryption target (think dm-crypt, geom eli, ...) compatible with Linux cryptsetup 2.7
initrd support Support for an initial ramdisk, a la Linux initramfs/initrd, which can be used to mount a supported FS from any volume as root mount (for example lvm, crypt and iscsi volumes) 2.7
udevd, libdevattr A device enumeration library and supporting daemon a la Linux libudev 2.7
opencrypto improvements involved in improving the performance of the software crypto (cryptosoft) by taking advantage of multiple CPU cores, as well as several fixes 2.7
